1. PANORAMICA GENERALE
Il progetto BLITZ è un'applicazione per il controllo di una troncatrice CNC a 2 teste. L'applicazione è sviluppata in Python utilizzando il framework Qt6 (PySide6) per l'interfaccia grafica.
Architettura del Sistema:
* Linguaggio principale: Python 3.x
* Framework UI: PySide6 (Qt6)
* Librerie principali: numpy (calcoli numerici), ortools (ottimizzazione)
* Database: SQLite per gestione commesse e tipologie

3. STRUTTURA DIRECTORY
La repository è organizzata nelle seguenti directory principali:

* qt6_app/: Applicazione principale Qt6
* qt6_app/ui_qt/: Moduli interfaccia utente
* qt6_app/ui_qt/data/: Gestione database e persistenza dati
* qt6_app/ui_qt/dialogs/: Finestre di dialogo modali
* qt6_app/ui_qt/logic/: Logica di business (homing, planning, sequencing)
* qt6_app/ui_qt/pages/: Pagine principali dell'applicazione
* qt6_app/ui_qt/services/: Servizi e motori (calcoli, import DXF, parametrico)
* qt6_app/ui_qt/utils/: Utility e configurazioni
* qt6_app/ui_qt/widgets/: Componenti UI riutilizzabili
* ui/: Moduli condivisi (stato macchina)
* data/: File di configurazione e dati
* data/typologies/: Definizioni tipologie prodotti

5. MODULI E COMPONENTI PRINCIPALI
5.1 Interfaccia Utente (UI)
L'interfaccia è costruita con PySide6 e organizzata in pagine, dialog e widget. Le pagine principali includono: Home, Manuale, Automatico, Semi-automatico, Tipologie, QuoteVani e Utility.
5.2 Gestione Dati
Il sistema utilizza SQLite per la persistenza. I DAO (Data Access Objects) gestiscono le operazioni su commesse e tipologie. Include funzionalità di seed per popolare il database.
5.3 Logica di Controllo
Moduli dedicati per homing (azzeramento macchina), planning (pianificazione tagli), refining (ottimizzazione), e sequencing (sequenziamento operazioni).
5.4 Servizi
Include motori per calcoli parametrici, import DXF, integrazione QCAD, gestione RS485, e store per ordini, profili e tipologie.
5.5 Hardware e Machine State
Il modulo machine_state gestisce lo stato della macchina CNC: posizionamento, angoli teste, encoder, freno, frizione, emergenza, e code di lavoro.


6. FLUSSI OPERATIVI PRINCIPALI
6.1 Modalità Manuale
- L'operatore controlla direttamente posizione e angoli
- Gestione freno e frizione manuale
- Pulsanti teste per esecuzione tagli
6.2 Modalità Semi-automatica
- Impostazione parametri di taglio
- Posizionamento automatico
- Conteggio pezzi programmato
6.3 Modalità Automatica
- Caricamento commesse da database
- Ottimizzazione sequenza tagli
- Esecuzione automatica con monitoraggio
6.4 Gestione Tipologie
- Editor tipologie parametriche
- Calcolo automatico dimensioni componenti
- Export e import configurazioni
